# 13 项目组推荐流程与速查表

## 1. 这一节要解决什么问题

这一节给你一份"每次开始工作都可以照着走"的完整推荐流程，以及一份随时查阅的常用命令速查表，让你不需要每次都去翻文档。

## 2. 日常安全工作流

### 2.1 每次开始工作的标准流程

```powershell
# ============ 第一步：确认当前状态 ============
git status
# 确认工作区干净，确认当前在哪个分支

# ============ 第二步：切换到 main 并更新 ============
git switch main
git pull
# 拉取最新代码，从最新状态出发

# ============ 第三步：新建功能分支 ============
git switch -c feat/weekly-tanglx-w26
# 分支名格式：类型/描述-作者标识

# ============ 第四步：做你的工作 ============
# 修改文件，编写内容...

# ============ 第五步：查看状态，确认改了什么 ============
git status
git diff 文件名  # 可选，看具体改了哪些行

# ============ 第六步：添加文件到暂存区 ============
git add docs\weekly\2026-W26-tanglx.md
# 推荐指定文件名，而不是 git add .

git status  # 再次确认暂存区内容

# ============ 第七步：提交 ============
git commit -m "docs: 添加唐玲欣 2026-W26 周报"

# ============ 第八步：推送 ============
git push -u origin feat/weekly-tanglx-w26
# 第一次 push 新分支用 -u，之后直接 git push

# ============ 第九步：在 GitHub 创建 PR ============
# 打开 GitHub 仓库页面
# 点击 "Compare & pull request" 或手动创建 PR
# 填写标题和描述
```

### 2.2 以写周报为例的完整示例

场景：提交 2026 年第 26 周的周报，分支名 `feat/weekly-tanglx-w26`

```powershell
# 确认起点
git switch main
git pull

# 输出示例：
# Already up to date.

# 新建分支
git switch -c feat/weekly-tanglx-w26

# 新建周报文件
New-Item docs\weekly\2026-W26-tanglx.md

# 用编辑器编写周报内容...（在 VS Code 里编辑）

# 确认状态
git status

# 输出：
# On branch feat/weekly-tanglx-w26
# Untracked files:
#         docs/weekly/2026-W26-tanglx.md

# 添加文件
git add docs\weekly\2026-W26-tanglx.md

# 确认进入暂存区
git status

# 输出：
# Changes to be committed:
#         new file:   docs/weekly/2026-W26-tanglx.md

# 提交
git commit -m "docs: 添加唐玲欣 2026-W26 周报"

# 输出：
# [feat/weekly-tanglx-w26 3a7b1d9] docs: 添加唐玲欣 2026-W26 周报
#  1 file changed, 45 insertions(+)
#  create mode 100644 docs/weekly/2026-W26-tanglx.md

# 推送
git push -u origin feat/weekly-tanglx-w26

# 输出：
# Branch 'feat/weekly-tanglx-w26' set up to track remote branch from 'origin'.

# 打开 GitHub，创建 PR，等老师 review
```

### 2.3 PR merge 后的清理

```powershell
# PR 合并后，同步最新 main
git switch main
git pull

# 删除已完成的功能分支（本地）
git branch -d feat/weekly-tanglx-w26

# 如果 GitHub 上也要删除远端分支（可选，GitHub 上 merge 后也有删除按钮）
git push origin --delete feat/weekly-tanglx-w26
```

## 3. 速查表

### 3.1 查看状态类

| 场景 | 命令 |
| --- | --- |
| 查看当前工作区状态 | `git status` |
| 查看简洁状态 | `git status -s` |
| 查看工作区 vs 暂存区差异 | `git diff` |
| 查看暂存区 vs 最近 commit 差异 | `git diff --staged` |
| 查看远端地址 | `git remote -v` |
| 查看 Git 配置 | `git config --list` |

### 3.2 分支操作类

| 场景 | 命令 |
| --- | --- |
| 查看本地所有分支 | `git branch` |
| 查看所有分支（含远端） | `git branch -a` |
| 切换到已有分支 | `git switch 分支名` |
| 新建并切换到新分支 | `git switch -c 分支名` |
| 从指定分支新建分支 | `git switch -c 分支名 origin/main` |
| 删除本地分支（安全） | `git branch -d 分支名` |
| 删除远端分支 | `git push origin --delete 分支名` |

### 3.3 提交类

| 场景 | 命令 |
| --- | --- |
| 添加指定文件到暂存区 | `git add 文件名` |
| 添加所有修改（谨慎） | `git add .` |
| 从暂存区取出文件 | `git restore --staged 文件名` |
| 提交 | `git commit -m "type: 描述"` |
| 查看最近 5 条提交（简洁） | `git log --oneline -5` |
| 图形化查看所有分支历史 | `git log --oneline --graph --all` |

### 3.4 同步类

| 场景 | 命令 |
| --- | --- |
| 拉取并合并最新代码 | `git pull` |
| 只下载不合并 | `git fetch origin` |
| 第一次 push 新分支 | `git push -u origin 分支名` |
| 后续 push（已设追踪） | `git push` |
| 查看本地比远端多什么 | `git log origin/main..HEAD --oneline` |

### 3.5 撤销类（安全做法）

| 场景 | 命令 | 是否安全 |
| --- | --- | --- |
| 把文件从暂存区取出（不丢修改） | `git restore --staged 文件名` | ✅ 安全 |
| 临时保存未提交改动 | `git stash` | ✅ 安全 |
| 恢复 stash | `git stash pop` | ✅ 安全 |
| 丢弃单文件工作区修改 | `git restore 文件名` | ⚠️ 确认后用 |
| 撤销最近一次 commit，保留修改到暂存区 | `git reset --soft HEAD~1` | ⚠️ 未 push 时用 |
| 撤销已 push 的提交 | `git revert commit_id` | ✅ 适合共享历史 |
| 丢弃所有未提交修改 | `git reset --hard HEAD` | ⛔ 新手慎用 |

### 3.6 配置与初始化类

| 场景 | 命令 |
| --- | --- |
| 配置用户名 | `git config --global user.name "名字"` |
| 配置邮箱 | `git config --global user.email "邮箱"` |
| 克隆仓库（SSH） | `git clone git@github.com:用户名/仓库名.git` |
| 克隆仓库（HTTPS） | `git clone https://github.com/用户名/仓库名.git` |
| 测试 SSH 连接 | `ssh -T git@github.com` |
| 生成 SSH Key | `ssh-keygen -t ed25519 -C "邮箱"` |
| 查看公钥 | `cat ~/.ssh/id_ed25519.pub` |
| 初始化本地仓库 | `git init` |
| 添加远端 origin | `git remote add origin 仓库地址` |
| 首次推送 main | `git push -u origin main` |

### 3.7 进阶命令类

| 场景 | 命令 |
| --- | --- |
| 临时保存工作进度 | `git stash push -m "说明"` |
| 查看 stash 列表 | `git stash list` |
| 恢复最近 stash | `git stash pop` |
| 查看某文件的修改历史 | `git log --oneline -- 文件名` |
| 检查文件是否被 ignore | `git check-ignore -v 文件名` |
| 移除已追踪文件（不删本地） | `git rm --cached 文件名` |
| 打版本 tag | `git tag -a v1.0.0 -m "说明"` |
| 推送 tag 到远端 | `git push origin v1.0.0` |
| 查看本地操作记录 | `git reflog` |
| 把某个 commit 拿到当前分支 | `git cherry-pick commit_id` |
| 添加上游仓库 | `git remote add upstream 仓库地址` |
| 获取上游更新 | `git fetch upstream` |

### 3.8 GitHub 协作类

| 场景 | 做法 |
| --- | --- |
| PR 关联并自动关闭 Issue | 在 PR 描述写 `Closes #编号` |
| PR 只说明相关 Issue | 在 PR 描述写 `Related to #编号` |
| 请求别人审查 PR | 在 GitHub PR 页面添加 Reviewer |
| 标记任务负责人 | 给 Issue 或 PR 设置 Assignee |
| PR 检查失败 | 点开 failed check 的 Details 看 Actions 日志 |
| fork 项目后同步原仓库 | 添加 `upstream`，再 `git fetch upstream` |

## 6. 危险命令速查（提醒自己不要随便用）

| 命令 | 危险原因 | 安全替代 |
| --- | --- | --- |
| `git push -f` | 覆盖远端历史，破坏他人工作 | 先 `git pull` |
| `git reset --hard` | 丢弃所有未提交改动，不可恢复 | `git stash` |
| `git clean -fd` | 删除未追踪文件，不可恢复 | 先 `git clean -fdn` 预览 |
| 直接 push main | 绕过 review，影响所有人 | 用 PR 流程 |

## 7. 常见报错速查

| 报错关键词 | 原因 | 处理 |
| --- | --- | --- |
| `Permission denied (publickey)` | SSH Key 未配置或未添加到 GitHub | 查 02 节配置 SSH |
| `not a git repository` | 当前目录不是 Git 仓库 | `cd` 进正确目录 |
| `rejected (non-fast-forward)` | 远端有新提交，push 被拒 | 先 `git pull` 再 push |
| `CONFLICT` | 合并冲突，需要手动解决 | 编辑冲突文件，add，commit |
| `local changes would be overwritten` | 工作区有未提交改动，pull 被阻止 | 先 `git stash` 再 pull |
| `detached HEAD` | HEAD 没有指向分支 | `git switch main` 切回分支 |
| `protected branch hook declined` | main 分支受保护，禁止直接 push | 新建分支并走 PR |
| `failed status checks` | PR 自动检查失败 | 打开 Actions 日志定位失败步骤 |
| `refusing to merge unrelated histories` | 本地和远端是两段独立历史 | 初始化场景才考虑 `--allow-unrelated-histories` |
