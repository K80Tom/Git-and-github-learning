# Git 与 GitHub 协作学习资料

这套资料面向基础较弱的学习者，目标是让你从"照着命令操作"走到"理解每一步为什么这样做，能安全完成日常团队协作，遇到报错知道怎么判断"。

建议阅读顺序：

```text
00_学习总路线.md
01_Git与GitHub是什么.md
02_配置Git与SSH.md
03_克隆仓库与理解Remote.md
04_三个区域与工作流程.md
05_分支为什么存在.md
06_拉取与同步代码.md
07_提交commit规范.md
08_推送与Pull_Request.md
09_看懂git_status与git_log.md
10_常见报错与排查.md
11_危险命令与安全习惯.md
12_进阶概念补充.md
13_项目组推荐流程与速查表.md
14_从零创建仓库与首次推送.md
15_撤销恢复与误操作补救.md
16_GitHub协作功能与分支保护.md
17_GitHub_Actions与PR检查.md
18_Fork与开源贡献流程.md
```

每个文件都能单独阅读，也能连成完整学习体系。你可以先按 `00_学习总路线.md` 的 7 天计划快速跑一遍，再用 14 天计划补扎实。

阅读建议：主线 00→13 按顺序走；`11_危险命令` 建议在开始操作前提前看一遍；`13_速查表` 随时当字典翻。14→18 是补充篇，适合在跑通日常协作后继续学习：从零建仓库、误操作恢复、GitHub 团队功能、Actions 检查和开源 fork 流程。
